# Contribuindo

Obrigado por considerar contribuir!

## Fluxo

1. Faça um fork do repositório.
2. Crie uma branch a partir de `main`.
3. Faça commits pequenos e descritivos (Conventional Commits é recomendado).
4. Garanta que lint e testes passam localmente:
   - `npm run lint`
   - `npm test`
5. Abra um Pull Request.

## Padrões

- Mantenha o código simples e legível.
- Atualize documentação quando necessário.
- Evite mudanças breaking; se inevitáveis, documente claramente.

## Reportando bugs

Use o template de issue e inclua passos para reproduzir, versão do Node.js e logs.
